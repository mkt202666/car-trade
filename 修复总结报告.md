### 修复1: 统一前后端接口命名

#### 问题分析
前端`src/api/ai.js`中存在重复方法:
- `marketAnalysis` 和 `getMarketAnalysis` (都指向`POST /ai/market-analysis`)
- `aiSearch` 和 `searchAllCars` (都指向`POST /ai/search`)

#### 修复方案
**文件**: `car-trade-frontend/src/api/ai.js`

**修改内容**:
1. 删除重复的`getMarketAnalysis`(第17行)
2. 删除重复的`searchAllCars`(第20行)
3. 新增`generateCustomerCopywriting`方法(获客文案生成)

**修改后的API列表**:
```javascript
export const marketAnalysis = (data) => post('/ai/market-analysis', data)
export const aiSearch = (data) => post('/ai/search', data)
export const generateCopywriting = (data) => post('/ai/copywriting', data)
export const generateCustomerCopywriting = (data) => post('/ai/customer-generation', data) // 新增
export const autoOutreach = (data) => post('/ai/auto-outreach', data)
export const distributeCar = (data) => post('/ai/distribute', data)
export const aiChat = (data) => post('/ai/chat', data)
export const carAnalysis = (data) => post('/ai/car-analysis', data)

export const getMyCars = (params) => uni.http.get('/cars', { params })
export const getCompetitors = (params) => uni.http.get('/ai/competitors', { params })
export const getSuggestions = (params) => uni.http.get('/ai/suggestions', { params })
```

#### 更新引用
**受影响文件**:
1. `car-trade-frontend/src/pages/ai-search-car/index.vue`
   - 将`searchAllCars`替换为`aiSearch`
   
2. `car-trade-frontend/src/pages/ai-market-report/index.vue`
   - 将`getMarketAnalysis`替换为`marketAnalysis`

**影响范围**: 2个页面文件,4处调用点

### 修复2: 同步前后端权限配置

#### 问题分析
对比`WebMvcConfig.java`和`permissions.js`发现:
1. 前端路径错误: `/ai/generate-copywriting` → 实际应为`/ai/copywriting`
2. 后端缺少排除项: 
   - `POST /users/sms/send` (短信发送)
   - `GET /users/{id}` (用户公开信息)
   - `/api/v1/brands/**` (品牌数据)

#### 修复方案

##### 前端修复
**文件**: `car-trade-frontend/src/api/permissions.js`

**修改CERTIFIED_ENDPOINTS**(第52-63行):
```javascript
// 修改前
{ path: /^\/ai\/generate-copywriting$/, method: 'POST' },

// 修改后
{ path: /^\/ai\/copywriting$/, method: 'POST' },
```

##### 后端修复
**文件**: `car-trade-backend/src/main/java/com/pancosky/newcartrade/config/WebMvcConfig.java`

**修改excludePathPatterns**(第51-93行):
```java
.excludePathPatterns(

    // 新增: 品牌车系车型
    "/api/v1/brands",
    "/api/v1/brands/**",
    
    // ... 其他配置 ...
)
```

## 五、注意事项

### 已知限制
1. **copywriting vs customer-generation**
   - `/ai/copywriting`: 车源文案生成(面向车辆描述)
   - `/ai/customer-generation`: 获客文案生成(面向营销推广)
   - 两者功能相似但场景不同,需在后端区分实现

2. **权限边界**
   - 短信发送接口已开放给未登录用户,需注意防刷机制
   - 建议后端增加IP频率限制或图形验证码

### 后续优化建议
1. **API规范化**
   - 考虑统一API命名规范(如全部使用动词+名词格式)
   - 建立API文档自动生成机制

2. **权限管理**
   - 考虑引入权限配置中心,避免前后端手动同步
   - 增加权限变更审计日志

3. **功能增强**
   - 获客文案可增加模板选择(朋友圈、抖音、小红书等)
   - 支持文案风格定制(专业、幽默、温馨等)
