# 运营端功能清单文档

> 基于原型页面 `http://shengtaiprd.pancosky.com/5dadmin/` 的全面分析  
> 分析日期：2026-06-13  
> 原型版本：v3.5

---

## 一、项目概述

**5D好车 B2B二手车交易平台运营端** 是面向平台运营人员的管理后台，用于对平台的用户、车行、车源、交易、资金等核心业务进行全方位管理与监控。

- **原型地址**：`http://shengtaiprd.pancosky.com/5dadmin/`
- **技术栈**：React + Vite + Tailwind CSS + Ant Design 5.x（暗色主题）
- **登录账号**：`yuan2026@5d.com`（超级合规风控管理员 AD）

---

## 二、整体布局结构

运营端采用经典的 **左侧导航 + 顶部栏 + 主内容区** 布局：

```
┌──────────────────────────────────────────────────────────┐
│  顶部栏：Logo + 系统名称 + 登录信息 + 还原预设按钮      │
├──────────┬───────────────────────────────────────────────┤
│          │                                               │
│  左侧     │  主内容区（各功能模块页面）                    │
│  导航栏   │                                               │
│  (11个   │                                               │
│  模块)   │                                               │
│          │                                               │
└──────────┴───────────────────────────────────────────────┘
│  底部合规声明区                                          │
└──────────────────────────────────────────────────────────┘
```

### 左侧导航栏模块（共11个）

### 3.1 分析仪表盘（Dashboard）

**页面功能**：展示平台核心运营数据的关键指标与趋势分析

#### 界面布局
- **顶部**：页面标题「分析仪表盘」+ 简要描述
- **KPI卡片区**：一行展示多个核心指标卡片
- **图表区**：折线图（趋势）+ 饼图（占比）
- **右侧/下方**：审批队列 + 预警系统

#### 数据展示组件
| 组件类型 | 说明 |
|---------|------|
| KPI卡片 | 展示总计数值（用户总数、车源总数、交易总数、待审核数等） |
| 折线图 | 每日交易趋势（日维度） |
| 饼图 | 车源渠道分布 |
| 优惠券统计 | 优惠券发放/使用统计 |
| 审批队列 | 待审批事项列表（车行审核、交易审核等） |
| 预警系统 | 异常预警信息（保证金不足、违规提醒等） |

## 五、数据流转关系

### 5.1 移动端与运营端数据关系

```
移动端（uni-app）                   运营端（React）
┌──────────────┐                 ┌──────────────┐
│  用户注册     │──────┬────────→│ 用户管理     │
│  车行入驻申请 │──────┼────────→│ 车行注册审核 │
│  车源发布    │──────┼────────→│ 5D车源管理   │
│  求购发布    │──────┼────────→│ 求购管理     │
│  交易下单    │──────┼────────→│ 交易管理     │
└──────────────┘      │         └──────────────┘
                       │
┌──────────────┐      │         ┌──────────────┐
│  首页Banner  │←─────┼────────│ 资源管理     │
│  弹窗广告    │←─────┼────────│ (Banner配置)  │
│  用户协议    │←─────┼────────│ (协议文本)    │
└──────────────┘      │         └──────────────┘
                       │
┌──────────────┐      │
│  车行保证金  │←─────┼────────→│ 保证金现金流 │
└──────────────┘      │
```

### 5.2 核心数据流

1. **车行审核流程**：移动端提交入驻申请 → 运营端审核 → 审核通过后车行可发布车源
2. **车源管理流程**：车行发布车源 → 运营端可下架/删除违规车源
3. **交易管理流程**：移动端下单 → 运营端可介入处理纠纷/退款
4. **保证金流程**：交易成功扣除佣金 → 保证金变动 → 运营端记录流水
5. **资源配置流程**：运营端配置Banner/协议 → 移动端读取并展示
