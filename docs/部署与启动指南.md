## 二、配置文件说明

项目采用 Spring Boot 多环境配置，共 4 个配置文件：

```
src/main/resources/
├── application.yml           # 基础配置（公共）
├── application-local.yml     # 本地开发环境
├── application-test.yml      # 测试环境
└── application-prod.yml      # 生产环境
```

### 配置文件职责

| 文件 | 内容 |
|------|------|
| `application.yml` | 服务端口、连接池、Jackson、MyBatis-Plus、AI配置等公共配置 |
| `application-local.yml` | 本地数据库/Redis/RocketMQ 地址和密码（硬编码），DEBUG 日志 |
| `application-test.yml` | 测试环境配置，通过环境变量注入敏感信息 |
| `application-prod.yml` | 生产环境配置，所有敏感信息必须通过环境变量注入 |

### 切换环境

通过 `spring.profiles.active` 参数切换，默认为 `local`：

```bash
# 方式一：启动参数
java -jar app.jar --spring.profiles.active=prod

# 方式二：环境变量
export SPRING_PROFILES_ACTIVE=prod
java -jar app.jar

# 方式三：JVM 参数
java -Dspring.profiles.active=prod -jar app.jar
```

## 三、数据库初始化

### 1. 创建数据库

```sql
CREATE DATABASE new_car_trade WITH ENCODING 'UTF8' LC_COLLATE 'zh_CN.utf8' LC_CTYPE 'zh_CN.utf8' TEMPLATE template0;
```

### 2. 初始化表结构和数据

`car-trade-backend/src/main/resources/` 目录下共有三个 SQL 脚本，**全部使用 `WHERE NOT EXISTS` / `IF NOT EXISTS` 幂等语法**，可安全重复执行。

| 文件 | 定位 | 内容概览 | 依赖 |
| --- | --- | --- | --- |
| `init.sql` | 完整初始化脚本 | 建库、31 张数据表（含索引/注释）、测试用户 / 品牌 / 车系 / 车型 / 车源 / 车源图片 / 会员套餐 | 空库即可执行 |
| `data-cities.sql` | 行政区划数据 | 34 个省级行政区、333 个地级行政区、部分区县（共约 386 条 INSERT），用于城市筛选 / 发布车源时选择城市 | 依赖 `init.sql` 已创建 `cities` 表 |
| `schema.sql` | 增量迁移与补充数据 | 为 `car_sources` 补充出口城市字段；幂等 INSERT 一批测试品牌、车系、车型、用户、车源、车源图片；重置相关序列 | 依赖 `init.sql` 已创建所有基础表 |

**推荐执行顺序（按依赖）**：

```bash
# 1) init.sql —— 建库建表 + 基础测试数据（你已手动完成）
psql -h <host> -U postgres -d new_car_trade -f car-trade-backend/src/main/resources/init.sql

# 2) data-cities.sql —— 行政区划数据（城市选择器所需）
psql -h <host> -U postgres -d new_car_trade -f car-trade-backend/src/main/resources/data-cities.sql

# 3) schema.sql —— 增量迁移 + 补充测试数据（缺车源出口城市/图片时执行）
psql -h <host> -U postgres -d new_car_trade -f car-trade-backend/src/main/resources/schema.sql
```

**前置条件**：
- PostgreSQL 16 已安装，已创建数据库 `new_car_trade`
- 执行账号拥有 `CONNECT`、`CREATE TABLE`、`INSERT`、`ALTER`、`SELECT` 权限
- 若 `init.sql` 已手动执行过，`data-cities.sql` 与 `schema.sql` 可直接独立执行，互不依赖、不会重复插入

**启动时自动行为（重要）**：
- `spring.sql.init.mode=never` —— Spring Boot 原生 SQL 初始化已**禁用**
- `StartupRunner`（`car-trade.init-db=true` 时触发）—— 仅执行少量 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 与 `CREATE TABLE IF NOT EXISTS` 语句（用于 `chat_messages` 等补表），不会读取任何外部 `.sql` 文件
- 因此**城市数据与测试车源数据必须通过上面的 `psql -f` 手动执行**

## 四、后端启动

### 1. 编译打包

```bash
cd car-trade-backend
mvn clean package -DskipTests
```

### 2. 本地开发启动

```bash
# 方式一：Maven 直接运行
mvn spring-boot:run

# 方式二：运行 jar（默认 local 环境）
java -jar target/new-car-trade-1.0.0-SNAPSHOT.jar
```

### 3. 测试环境启动

```bash
java -jar target/new-car-trade-1.0.0-SNAPSHOT.jar --spring.profiles.active=test
```

### 4. 生产环境启动

```bash
# 设置环境变量
export DB_HOST=your-db-host
export DB_NAME=new_car_trade
export DB_USERNAME=postgres
export DB_PASSWORD=your-db-password
export REDIS_HOST=your-redis-host
export REDIS_PASSWORD=your-redis-password
export JWT_SECRET=your-jwt-secret-key-at-least-32-chars
export ROCKETMQ_NAMESRV=your-rocketmq-nameserver
export ROCKETMQ_AK=your-access-key
export ROCKETMQ_SK=your-secret-key
export ROCKETMQ_NS=your-namespace
export AI_ENABLED=true
export AI_API_KEY=your-ai-api-key

# 启动
java -jar target/new-car-trade-1.0.0-SNAPSHOT.jar --spring.profiles.active=prod
```

### 5. 后台运行（生产推荐）

```bash
nohup java -jar target/new-car-trade-1.0.0-SNAPSHOT.jar --spring.profiles.active=prod > app.log 2>&1 &
```

## 八、常见问题

### 1. 数据库连接失败
检查 `DB_HOST`、`DB_PORT`、`DB_USERNAME`、`DB_PASSWORD` 是否正确，数据库是否已创建。

### 2. Redis 连接超时
检查 `REDIS_HOST`、`REDIS_PASSWORD` 是否正确，Redis 服务是否可访问。

### 3. RocketMQ 连接失败
检查 `ROCKETMQ_NAMESRV`、`ROCKETMQ_AK`、`ROCKETMQ_SK` 是否正确。RocketMQ 为可选组件，连接失败不影响核心功能。

### 4. SQL 初始化报错（表已存在 / 主键冲突）

- `spring.sql.init.mode=never` —— Spring Boot 原生 SQL 初始化已**禁用**。
- `StartupRunner` 只执行 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` / `CREATE TABLE IF NOT EXISTS`，遇到重复会自动跳过并打印 WARN 日志。
- `init.sql` / `data-cities.sql` / `schema.sql` 全部使用 `WHERE NOT EXISTS` 或 `IF NOT EXISTS`，重复执行**不会**导致数据重复。
- 如遇到 `duplicate key value violates unique constraint`，通常是之前未用幂等脚本插入过数据；先清理冲突数据或只执行尚未运行的脚本。

### 5. 城市选择器没有数据 / 车源没有出口城市字段

说明 `data-cities.sql`（行政区划数据）或 `schema.sql`（补充车源出口字段）未执行。在启动应用前执行：

```bash
psql -h <host> -U postgres -d new_car_trade -f car-trade-backend/src/main/resources/data-cities.sql
psql -h <host> -U postgres -d new_car_trade -f car-trade-backend/src/main/resources/schema.sql
```
