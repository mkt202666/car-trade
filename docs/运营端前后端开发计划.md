## 二、总体开发节奏

```
Week 1 (Day 1-5)          Week 2 (Day 6-12)         Week 3 (Day 13-18)
┌─────────────────┐      ┌─────────────────┐       ┌─────────────────┐
│ Phase 1: 基础    │      │ Phase 2: 业务    │       │ Phase 3: 辅助    │
│ 框架 + 认证     │─────▶│ 管理接口 + 前端  │──────▶│ 功能 + 联调测试  │

└─────────────────┘      └─────────────────┘       └─────────────────┘
```

| 阶段 | 工期 | 前端重点 | 后端重点 |
|------|------|---------|---------|
| **Phase 1** | 5 天 | Login 对接、Dashboard 数据渲染、API 层搭建 | 项目骨架、认证模块、Dashboard 聚合接口、用户查询 |
| **Phase 2** | 7 天 | 6 个业务页面对接 API（用户/车行/审核/车源/交易/保证金）| 21 个业务 CRUD 接口、Redis 事件发布、审计日志 |

| # | 方法 | URL | 说明 |
|---|------|-----|------|
| 17 | GET | `/api/v1/admin/shop-reviews` | 审核列表（status=PENDING/ALL）|
| 18 | PUT | `/api/v1/admin/shop-reviews/{id}/approve` | 审核通过 → 更新 users.certification_status |
| 19 | PUT | `/api/v1/admin/shop-reviews/{id}/reject` | 审核拒绝 → 填写拒绝原因 |

| 21 | GET | `/api/v1/admin/shop-reviews/pending-count` | 待审核数量（侧边栏徽标）|

**关键逻辑**：
- 审核通过 → `users.certification_status = CERTIFIED` + Redis 发布 `user:certification:{userId}` 事件
- 审核拒绝 → `users.certification_status = REJECTED` + 发送站内消息通知
- 批量审核 → 事务内批量更新 + 单次事件发布

## 七、关键依赖与风险

### 7.1 外部依赖

| 依赖项 | 说明 | 风险等级 |
|--------|------|---------|
| PostgreSQL | 共享移动端数据库，需提前确认表结构兼容性 | 🟡 中 |
| Redis | 共享实例，Pub/Sub 频道命名需与移动端协调 | 🟢 低 |
| JWT_SECRET | 两端共用同一个密钥，通过环境变量注入 | 🟢 低 |
| 文件上传（OSS）| Banner 图片/车型 Excel 导入需对接存储服务 | 🟡 中 |

### 7.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Dashboard 聚合查询慢 | 多表联查可能超时 | 预计算 + Redis 缓存 + 物化视图 |
| 并发编辑冲突 | 两端同时操作同一数据 | 乐观锁（version 字段）|
| 缓存一致性 | 运营端修改后移动端延迟感知 | Redis Pub/Sub 即时清除 |
| 大数据量分页 | 用户/车源/订单量增长后查询变慢 | 索引优化 + 游标分页备选 |

### 7.3 前后端约定

| 约定项 | 规范 |
|--------|------|
| API 前缀 | `/api/v1/admin` |
| 统一响应 | `{code: 0, message: "success", data: {...}, timestamp: "...", traceId: "..."}` |
| 分页响应 | `{code: 0, data: {list: [...], total: 100, page: 1, size: 20}}` |
| 错误码 | 0=成功, 400=参数错误, 401=未认证, 403=无权限, 404=不存在, 409=冲突, 500=服务器错误 |
| 日期格式 | `yyyy-MM-dd HH:mm:ss` |
| 金额精度 | Long 类型（分），前端展示时 ÷ 100 |
| ID 类型 | BIGSERIAL 自增 / BIGINT 雪花算法 |
| 分页参数 | `page`(从1开始) + `size`(默认20, 可选10/20/50/100) |
