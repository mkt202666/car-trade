## 一、联调前准备

### 1.1 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| JDK | 21 | `JAVA_HOME` 指向 temurin-21 |
| Maven | 3.9+ | 使用项目自带 `mvnw` |
| Node.js | 18+ | `npm run dev:h5` |
| PostgreSQL | 16 | 数据库 `new_car_trade` |
| Redis | 7 | 缓存/Token存储 |

### 1.2 启动顺序

```bash
# 1. 启动后端 (端口 8080)
cd car-trade-backend
./mvnw spring-boot:run

# 2. 启动前端 (端口 5173)
cd car-trade-frontend
npm run dev:h5

# 3. 验证后端健康
curl http://localhost:8080/api/v1/users/sms/send -X POST -H "Content-Type: application/json" -d '{"phone":"13800138000"}'
```

## 三、端到端业务流测试

### 流程1: 完整交易流程（P0）

```
车商A发布车源 → 车商B搜索找到 → 查看详情 → 联系卖家聊天
→ 付定金锁定 → 卖家确认 → 卖家补充合同 → 买家确认合同
→ 线下过户 → 买家确认收货 → 交易完成
```

| 步骤 | 操作 | 接口 | 验证点 |
|------|------|------|--------|
| 1 | 车商A登录 | `POST /users/login` | token获取成功 |
| 2 | 发布车源 | `POST /cars` | 车源状态=ACTIVE |
| 3 | 车商B登录 | `POST /users/login` | token获取成功 |
| 4 | 搜索车源 | `GET /cars?keyword=...` | 能找到该车源 |
| 5 | 查看详情 | `GET /cars/{id}` | 信息完整 |
| 6 | 联系卖家 | `POST /chat/conversations` | 会话创建 |
| 7 | 发消息 | `POST /chat/conversations/{id}/messages` | 消息发送成功 |
| 8 | 付定金 | `POST /orders` | 订单创建，保证金冻结 |
| 9 | 卖家确认 | `PUT /orders/{id}/confirm` | 状态=交易中 |
| 10 | 提交合同 | `POST /orders/{id}/contract` | 合同创建 |
| 11 | 买家确认 | `PUT /orders/{id}/contract/confirm` | 合同签署完成 |
| 12 | 完成交易 | `PUT /orders/{id}/complete` | 状态=已完成 |
| 13 | 验证余额 | `GET /finance/deposit` | 保证金已扣减 |
| 14 | 验证统计 | `GET /users/me/stats` | 成交数+1 |

### 流程2: 拍卖交易流程（P1）

```
卖家创建拍卖 → 买家关注 → 买家出价 → 拍卖倒计时结束
→ 系统结算 → 自动生成订单 → 进入常规交易流程
```

### 流程3: 争议处理流程（P1）

```
买家发起争议 → 提交凭证 → 等待客服介入 → 客服裁决
→ 退款/继续交易 → 争议关闭
```

### 流程4: 车行管理流程（P2）

```
车行老板邀请员工 → 员工扫码申请 → 老板审批通过
→ 员工可发布车源 → 车源显示车行标识
```
