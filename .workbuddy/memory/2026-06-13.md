## 修复问题（Phase 3 收尾）

### 修复的 Bug

1. **AdminAuthService 登录500** - `RedisTemplate<String, String>` 的 `increment()` 存储整数类型，
   `get()` 时 Integer 无法 cast 到 String。
   → 改用 `StringRedisTemplate`（Spring Boot 自动配置的 Bean）

2. **Dashboard Trend 500** - `AppOrderMapper.selectDailyTrend` 和 `AppUserMapper.selectDailyNewUsers`
   SQL 中 `created_at >= #{startDate}` 将 String 与 TIMESTAMP 比较，PostgreSQL 无隐式转换。
   → 参数加 `::timestamp` 显式类型转换

3. **Coupon Stats 500** - `coupons` 表无 `used_count` 列（实际有 `total_count` 和 `remain_count`）。
   → `COALESCE(SUM(total_count - remain_count), 0) as used_count`

4. **前端 Dashboard 数据不匹配** - 前端期望字段名（gmv/carNew/deposit等）与后端返回字段不一致，
   导致所有数据降级到 DEFAULT_KPI 硬编码值。
   → 修改前端 dashboard/index.jsx 正确映射后端字段；
   → 后端 DashboardKpiVO 新增 `gmv`/`deposit`/`depositActive`/`pendingProcessed` 字段并 Computing
