## 二、移动端用户类型判断规则

### 核心字段：`shop_name`

移动端通过 `users` 表的 `shop_name` 字段区分用户类型：

| 用户类型 | shop_name 值 | certification_status | 说明 |
|---------|-------------|---------------------|------|
| **个人用户** | NULL 或 '' | NONE/PENDING/CERTIFIED/REJECTED | 普通C端用户，可发布车源但无车行认证 |
| **车行用户** | 非空字符串 | PENDING/CERTIFIED/REJECTED | B端商家，提交车行认证申请后获得 |

### 判断逻辑（伪代码）

```java
public enum UserType {
    PERSONAL_USER,   // 个人用户
    SHOP_USER        // 车行用户
}

public UserType getUserType(User user) {
    if (user.getShopName() != null && !user.getShopName().isEmpty()) {
        return UserType.SHOP_USER;
    } else {
        return UserType.PERSONAL_USER;
    }
}
```

### 用户注册流程

1. **新用户注册** → `shop_name = NULL`, `certification_status = "NONE"`
2. **填写车行信息** → 设置 `shop_name = "某某二手车行"`
3. **提交认证申请** → `certification_status = "PENDING"`
4. **运营端审核通过** → `certification_status = "CERTIFIED"`

### 3.2 Service 层

**AdminUserService.listUsers()** - 根据 userType 动态构建查询条件

```java
public PageResult<AdminUserVO> listUsers(UserQueryDTO query) {
    String userType = query.getUserType();
    
    // 如果查询系统管理员，需要查询 admin_users 表
    if ("ADMIN_USER".equals(userType)) {
        return listAdminUsers(query);
    }
    
    // 默认查询移动端用户（users 表）
    LambdaQueryWrapper<AppUser> wrapper = new LambdaQueryWrapper<>();
    
    // 根据用户类型添加过滤条件
    if ("SHOP_USER".equals(userType)) {
        // 车行用户：shop_name 非空且非空字符串
        wrapper.isNotNull(AppUser::getShopName)
               .ne(AppUser::getShopName, "");
    } else if ("PERSONAL_USER".equals(userType)) {
        // 个人用户：shop_name 为空或空字符串
        wrapper.and(w -> w.isNull(AppUser::getShopName)
                        .or().eq(AppUser::getShopName, ""));
    } else if ("DELETED".equals(userType)) {
        // 已注销用户：deleted_at 非空
        wrapper.isNotNull(AppUser::getDeletedAt);
    } else {
        // ALL 或其他：只排除未删除的用户
        wrapper.isNull(AppUser::getDeletedAt);
    }
    
    // ... 其他过滤条件（关键字、状态、认证状态）
    
    Page<AppUser> page = appUserMapper.selectPage(...);
    return new PageResult<>(...);
}
```

### 3.3 系统管理员查询

**AdminUserService.listAdminUsers()** - 查询 `admin_users` 表

```java
private PageResult<AdminUserVO> listAdminUsers(UserQueryDTO query) {
    LambdaQueryWrapper<AdminUser> wrapper = new LambdaQueryWrapper<>();
    
    // 关键字搜索（用户名/昵称/邮箱）
    if (query.getKeyword() != null && !query.getKeyword().isEmpty()) {
        String kw = query.getKeyword();
        wrapper.and(w -> w.like(AdminUser::getUsername, kw)
                .or().like(AdminUser::getNickname, kw)
                .or().like(AdminUser::getEmail, kw));
    }
    
    // 状态过滤
    if (query.getStatus() != null && !query.getStatus().isEmpty()) {
        wrapper.eq(AdminUser::getStatus, query.getStatus());
    }
    
    Page<AdminUser> page = adminUserMapper.selectPage(...);
    
    // 转换为统一的 AdminUserVO（部分字段设为 null）
    List<AdminUserVO> voList = page.getRecords().stream().map(admin -> {
        AdminUserVO vo = new AdminUserVO();
        vo.setId(admin.getId());
        vo.setNickname(admin.getNickname() != null ? admin.getNickname() : admin.getUsername());
        vo.setRealName(null);      // 管理员无真实姓名
        vo.setPhone(null);         // 管理员无手机号
        vo.setShopName(null);      // 管理员无车行
        vo.setCertificationStatus(null);  // 管理员无认证状态
        vo.setStatus(admin.getStatus());
        vo.setDealCount(0);        // 管理员无交易
        vo.setOnSaleCount(0);      // 管理员无在售车源
        vo.setCreatedAt(admin.getCreatedAt());
        return vo;
    }).collect(Collectors.toList());
    
    return new PageResult<>(voList, page.getTotal(), ...);
}
```

## 六、数据库查询示例

### 车行用户

```sql
SELECT * FROM users 
WHERE deleted_at IS NULL
  AND shop_name IS NOT NULL 
  AND shop_name != ''
ORDER BY created_at DESC
LIMIT 20 OFFSET 0;
```

### 个人用户

```sql
SELECT * FROM users 
WHERE deleted_at IS NULL
  AND (shop_name IS NULL OR shop_name = '')
ORDER BY created_at DESC
LIMIT 20 OFFSET 0;
```

### 系统管理员

```sql
SELECT * FROM admin_users 
WHERE status = 'ACTIVE'
ORDER BY created_at DESC
LIMIT 20 OFFSET 0;
```

### 已注销用户

```sql
SELECT * FROM users 
WHERE deleted_at IS NOT NULL
ORDER BY deleted_at DESC
LIMIT 20 OFFSET 0;
```

## 七、注意事项

### 7.1 数据隔离

- **移动端用户** (`users` 表) 和 **运营端管理员** (`admin_users` 表) 保持独立
- 不合并用户体系，避免权限混淆
- 运营端仅作为统一视图展示，不修改底层数据结构

### 7.2 VO 转换

由于 `AppUser` 和 `AdminUser` 字段不完全一致，在 `listAdminUsers()` 中需要将管理员数据转换为统一的 `AdminUserVO`：

| 字段 | AppUser | AdminUser | 转换策略 |
|------|---------|-----------|---------|
| id | ✅ | ✅ | 直接使用 |
| nickname | ✅ | ✅ | 优先使用 nickname，否则用 username |
| phone | ✅ | ❌ | 管理员设为 null |
| shopName | ✅ | ❌ | 管理员设为 null |
| certificationStatus | ✅ | ❌ | 管理员设为 null |
| dealCount | ✅ |  | 管理员设为 0 |
| onSaleCount | ✅ | ❌ | 管理员设为 0 |

### 7.3 性能优化

- 当前实现为内存分页（先查全部再分页），大数据量时需改为数据库分页
- 考虑添加索引：
  ```sql
  CREATE INDEX idx_users_shop_name ON users(shop_name);
  CREATE INDEX idx_users_deleted_at ON users(deleted_at);
  CREATE INDEX idx_admin_users_status ON admin_users(status);
  ```

### 7.4 未来扩展

如需支持更多用户类型（如"IT开发"、"测试账号"），可以：

1. 在 `users` 表添加 `user_category` 字段
2. 或在 `UserQueryDTO` 中添加新的枚举值
3. 更新 Service 层的过滤逻辑
