## 五、潜在问题与风险

### 5.1 严重问题(Critical)

1. **❌ 图片上传功能缺失**
   - 影响: 车源发布、头像上传、证件上传等核心功能无法使用
   - 建议: 立即添加统一上传接口 `POST /api/v1/uploads`

2. **❌ 品牌数据硬编码**
   - 影响: 新增品牌/车系需前后端同时修改,维护成本高
   - 建议: 所有品牌相关页面改为调用后端API

3. **❌ WebSocket未集成**
   - 影响: 聊天消息、拍卖出价、订单状态变更无法实时推送
   - 建议: 集成SockJS + STOMP,监听 `/ws` 端点

### 5.2 重要问题(High)

4. **⚠️ Demo数据污染**
   - 位置: `pages/profile/index.vue`, `pages/message/index.vue`
   - 问题: 生产环境可能展示虚假数据
   - 建议: 仅在开发环境(`process.env.NODE_ENV === 'development'`)使用Demo数据

5. **⚠️ 表单验证不完整**
   - 位置: `pages/publish/index.vue`, `pages/purchase-demand/index.vue`
   - 问题: 缺少必填项校验、格式校验(如价格必须>0)
   - 建议: 添加统一表单验证工具类 `src/utils/validate.js`

6. **⚠️ 错误处理不一致**
   - 问题: 部分页面 `catch` 块只打印console,未给用户提示
   - 建议: 统一使用 `uni.$u.toast(error.message)` 提示用户

### 5.3 一般问题(Medium)

7. **⚠️ 分页逻辑不统一**
   - 问题: 部分页面用 `data.list`,部分用 `data.records`,部分用 `res.data`
   - 建议: 统一后端返回格式为 `{ code: 200, data: { list: [], total: 0 } }`

8. **⚠️ Token存储双重兼容**
   - 位置: `src/constants/storage.js`
   - 问题: 同时使用 `uni.setStorageSync` 和 `localStorage`,H5/小程序行为不一致
   - 建议: 封装统一存储工具,自动判断平台

9. **⚠️ 缺少全局Loading状态**
   - 问题: 每个页面单独管理 `loading` 变量,代码重复
   - 建议: 使用Vuex管理全局Loading状态

### 5.4 优化建议(Low)

10. **💡 缓存策略缺失**
    - 建议: 对城市列表、品牌列表等静态数据添加本地缓存(`uni.setStorageSync`)

11. **💡 图片懒加载**
    - 建议: 车源列表图片使用 `<lazy-load>` 或Intersection Observer API

12. **💡 骨架屏**
    - 建议: 首屏加载使用骨架屏(Skeleton Screen)提升体验

## 七、前端代码质量评估

### 7.1 优点

✅ **UI设计精美**: Liquid Glass风格,动画流畅,视觉层次清晰  
✅ **组件复用良好**: 自定义TabBar、通用按钮样式、统一颜色变量  
✅ **API封装规范**: 统一使用 `uni.http` 实例,拦截器处理token和错误  
✅ **权限控制完善**: `@RequiresAuth` 注解与前端 `permissions.js` 白名单对应  
✅ **响应式设计**: 适配不同屏幕尺寸(`@media` 查询)  
✅ **降级策略**: 城市加载失败时使用本地静态数据  

### 7.2 缺点

❌ **硬编码数据过多**: 品牌、城市、广告位等多处使用静态数据  
❌ **缺少单元测试**: 无任何测试文件(`src/__tests__` 目录为空)  
❌ **错误处理不一致**: 部分页面静默失败,未给用户提示  
❌ **表单验证薄弱**: 缺少统一的验证工具类  
❌ **代码重复**: 多个页面的 `fetchData` 逻辑相似,未抽取mixin/composable  
❌ **注释不足**: 复杂业务逻辑缺少中文注释  

### 7.3 技术债务

1. **Vue 2 Options API**: 项目使用Vue 2语法,迁移到Vue 3 Composition API可提升可维护性
2. **Vuex状态管理简单**: 只存储user/token,其他状态分散在各页面,应考虑模块化
3. **缺少TypeScript**: 大型项目建议使用TS增强类型安全
4. **构建配置简单**: `vite.config.js` 未优化(如代码分割、gzip压缩)
