## 🔴 P0级严重问题(必须立即修复)

### 1. 移动端 - 图片上传功能完全缺失
**影响页面**: `pages/publish/index.vue` (车源发布)  
**问题描述**: 
- 前端调用了`uni.chooseImage()`选择图片,但**未上传到服务器**
- 检测报告、证件材料的上传功能也未实现
- 导致用户无法真正发布车源

**修复方案**:
```javascript
// 需要补充的API
POST /api/v1/upload/image  // 单图上传
POST /api/v1/upload/multi  // 多图批量上传
POST /api/v1/upload/file   // 文件上传(检测报告/证件)
```

**工作量**: 2天(后端接口1天 + 前端对接1天)

---

### 2. 移动端 - 品牌/车系/车型数据硬编码
**影响页面**: `pages/publish/index.vue`, `pages/home/index.vue`  
**问题描述**:
- 前端维护了13个品牌的静态数据(`brandData`)
- 未调用后端`GET /api/v1/brands`等接口
- 导致品牌数据与后端不同步,新增品牌无法显示

**修复方案**:
```javascript
// 需要对接的API
GET /api/v1/brands                    // 品牌列表
GET /api/v1/brands/:id/series         // 车系列表
GET /api/v1/brands/series/:id/models  // 车型列表
```

**工作量**: 1天(前端改造)

### 5. 移动端 - VIN扫描/AI帮我填功能缺失
**影响页面**: `pages/publish/index.vue`  
**问题描述**:
- `scanVin()`方法只有toast提示,未集成OCR SDK
- `aiFill()`方法未实现,应调用AI接口自动填充表单
- 这两个是PRD中的亮点功能,目前完全是空壳

```javascript
- AI接口: POST /api/v1/ai/fill-car-info (基于图片/语音自动填充)
```

**工作量**: 3-5天(取决于OCR服务商接入难度)

### 9. 运营端 - 多个页面导出功能可能缺少JWT认证
**影响页面**: 
- `pages/user-manage/index.jsx`
- `pages/shop-manage/index.jsx`
- `pages/car-manage/index.jsx`
- `pages/trade-manage/index.jsx`
- `pages/dispute-manage/index.jsx`

**问题描述**:
- 5个页面使用`window.open(url)`直接访问导出URL
- **可能未携带JWT Token**,导致导出失败或安全风险

**修复方案**:
```javascript
// 改为通过API下载,携带Token
const response = await fetch(url, {
  headers: { Authorization: `Bearer ${token}` }
});
const blob = await response.blob();
// 触发下载
```

**工作量**: 1天
